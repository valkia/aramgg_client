# LOL Tips Client 需求规格文档

## 1. 概述

### 1.1 产品定位
LOL Tips Client 是一款英雄联盟ARAM（极地大乱斗）模式的辅助工具，为玩家提供海克斯强化（Augments）的实时胜率数据，帮助玩家在游戏中做出更优的海克斯选择。

### 1.2 核心价值
- 实时展示当前英雄对应各海克斯的胜率数据
- 在海克斯选择界面自动识别可选项并提供胜率参考
- 减少玩家查询外部数据的时间，提升游戏体验

### 2.2 功能一：英雄海克斯胜率弹窗

#### 2.2.1 触发条件
- 监控服务已启动
- 检测到用户选择了英雄（获取到有效的英雄ID）

#### 2.2.2 功能描述
当检测到用户选择英雄后，**自动弹出**海克斯胜率弹窗，展示该英雄对应所有海克斯的胜率排行。

#### 2.2.3 弹窗内容
| 数据项 | 说明 |
|--------|------|
| 英雄信息 | 当前选择的英雄名称和头像 |
| 海克斯列表 | 按胜率排序的海克斯列表 |
| 胜率数据 | 每个海克斯的胜率百分比 |
| 选取率 | 每个海克斯的选取率 |
| 稀有度 | 银色(Silver)、金色(Gold)、棱彩(Prismatic) |
| 场次数据 | 该海克斯的统计样本量 |

#### 2.2.4 UI 要求
- 弹窗位置：屏幕右侧悬浮
- 支持拖拽移动
- 透明度可调（避免遮挡游戏画面）
- 支持折叠/展开
- 可按稀有度筛选

#### 2.2.5 生命周期
- **显示时机**：检测到英雄ID后自动显示
- **隐藏条件**：
  - 用户手动关闭
  - 游戏选人阶段结束
  - 检测到游戏窗口关闭

#### 2.3.3 截图服务

| 参数 | 值 | 说明 |
|------|-----|------|
| 截图间隔 | 2-3秒 | 每2-3秒截图一次 |
| 启动延迟 | 10秒 | 检测到条件后等待10秒开始截图 |
| 截图范围 | 全屏 | 捕获整个游戏画面 |
| 存储位置 | ~/.lol-tips-client/screenshots/ | 临时存储，自动清理 |

#### 2.3.4 图像识别

##### 2.3.4.1 技术方案
- **OCR引擎**：Tesseract.js（本地识别，无网络依赖）
- **识别流程**：先检测界面特征 → 再进行OCR识别（避免无效调用）

##### 2.3.4.2 海克斯选择界面检测（特征检测）
在进行OCR之前，先通过图像特征检测判断当前截图是否为海克斯选择界面：

**检测特征（参考 `docs/demo.png`）：**
| 特征 | 说明 |
|------|------|
| 三卡片布局 | 屏幕中央并排显示3张卡片 |
| 卡片边框 | 紫色(史诗)/金色(传说)/棱彩 渐变边框 |
| 卡片尺寸比例 | 固定的宽高比约 1:1.4 |
| 卡片间距 | 三张卡片等间距排列 |
| 背景特征 | 半透明暗色遮罩 |

**检测算法：**
1. 颜色阈值检测 - 检测紫色/金色边框区域
2. 轮廓检测 - 识别三个矩形卡片轮廓
3. 位置验证 - 验证卡片位于屏幕中央区域
4. 置信度阈值 - 检测置信度 > 0.8 时才进行OCR

##### 2.3.4.3 海克斯名称 OCR
- **识别区域**：每张卡片的名称区域（卡片上部约1/4处）
- **识别内容**：海克斯中文名称
- **预处理**：灰度化、二值化、降噪
- **匹配方式**：与本地海克斯名称库（`augments-base.json`）进行模糊匹配
- **模糊匹配算法**：Levenshtein距离 或 Jaccard相似度

##### 2.3.4.4 卡片位置提取
在检测海克斯界面时，同时提取三张卡片的精确位置坐标：
```json
{
  "cards": [
    { "x": 320, "y": 200, "width": 280, "height": 400 },
    { "x": 640, "y": 200, "width": 280, "height": 400 },
    { "x": 960, "y": 200, "width": 280, "height": 400 }
  ]
}
```

##### 2.3.4.5 识别结果
```json
{
  "detected": true,
  "confidence": 0.92,
  "augments": [
    { "name": "收伤", "id": 1xxx, "confidence": 0.95, "position": { "x": 320, "y": 200 } },
    { "name": "战争交响乐", "id": 1xxx, "confidence": 0.92, "position": { "x": 640, "y": 200 } },
    { "name": "任务：海牛阿浆的勇士", "id": 1xxx, "confidence": 0.88, "position": { "x": 960, "y": 200 } }
  ]
}
```

#### 2.3.5 胜率浮窗

##### 2.3.5.1 浮窗定位方式
**跟随卡片位置显示**：浮窗根据图像识别获取的卡片坐标，精确定位在每张海克斯卡片上方或下方。

```
┌─────────────────────────────────────────────────────────┐
│                      游戏画面                            │
│                                                         │
│     ┌──────┐         ┌──────┐         ┌──────┐         │
│     │52.3% │         │48.7% │         │55.1% │ ← 胜率浮窗│
│     │ 推荐 │         │      │         │ 最佳 │         │
│     └──────┘         └──────┘         └──────┘         │
│     ┌──────┐         ┌──────┐         ┌──────┐         │
│     │      │         │      │         │      │         │
│     │ 卡片1│         │ 卡片2│         │ 卡片3│ ← 海克斯卡片│
│     │      │         │      │         │      │         │
│     └──────┘         └──────┘         └──────┘         │
│                                                         │
└─────────────────────────────────────────────────────────┘
```

##### 2.3.5.2 单个浮窗内容
每张卡片对应一个小浮窗：
| 元素 | 说明 |
|------|------|
| 胜率 | 大字体显示，如 "52.3%" |
| 推荐标签 | 胜率最高标记"最佳"，次高标记"推荐" |
| 样本量 | 小字显示场次，如 "5.2k场" |
| 背景色 | 根据胜率高低渐变（绿色高、红色低） |

##### 2.3.5.3 推荐指数计算
推荐指数基于以下因素综合计算：
- 胜率 (win_rate) - 权重 60%
- 选取率 (pick_rate) - 权重 20%
- 样本量 (num_games) - 权重 20%

```javascript
// 推荐指数公式
score = winRate * 0.6 + pickRate * 0.2 + min(numGames/1000, 1) * 0.2
```

##### 2.3.5.4 UI 要求
- **定位方式**：跟随卡片位置，显示在卡片正上方
- **偏移量**：卡片顶部向上偏移 60px
- **浮窗尺寸**：与卡片等宽，高度约 50px
- **透明度**：85% 不透明
- **边框**：与卡片稀有度颜色一致
- **高亮效果**：胜率最高的浮窗添加发光效果

##### 2.3.5.5 适配不同分辨率
| 分辨率 | 卡片位置计算 |
|--------|-------------|
| 1920x1080 | 基准分辨率，直接使用识别坐标 |
| 2560x1440 | 坐标按比例 1.33 缩放 |
| 3840x2160 | 坐标按比例 2.0 缩放 |

##### 2.3.5.6 生命周期
- **显示时机**：OCR 识别到海克斯选择界面后立即显示
- **位置更新**：每次截图分析后更新位置（处理游戏窗口移动情况）
- **关闭条件**：
  - 用户选择海克斯后自动关闭（检测到界面消失）
  - 连续3次截图未检测到海克斯界面
  - 用户手动关闭

### 2.4 监控服务

#### 2.4.1 英雄选择监控

| 参数 | 值 | 说明 |
|------|-----|------|
| 轮询间隔 | 2秒 | 每2秒检查一次 |
| 数据源 | LCU API | 通过League Client API获取 |

- 接口：`/lol-champ-select/v1/session`
- 认证：从游戏目录的 lockfile 获取
- 返回数据：包含 `myTeam` 数组，其中包含英雄ID

#### 2.4.2 游戏窗口监控

| 参数 | 值 | 说明 |
|------|-----|------|
| 轮询间隔 | 1秒 | 每秒检查一次 |
| 检测目标 | League of Legends.exe 窗口 | 检测游戏进程窗口 |
| 检测内容 | 窗口是否存在、是否前台 | 判断游戏状态 |

## 4. 非功能需求

### 4.1 性能要求
- 截图捕获延迟 < 100ms
- 特征检测延迟 < 200ms
- OCR 识别延迟 < 800ms
- 单次完整识别流程 < 1.5s
- 内存占用 < 300MB（含 Tesseract 模型）
- CPU 使用率（空闲时）< 5%
- CPU 使用率（识别时）< 30%

### 4.2 兼容性
- Windows 10/11
- 英雄联盟国服客户端
- 支持多显示器配置
- 支持不同分辨率（1080p、1440p、4K）

### 4.3 安全性
- 不修改游戏文件
- 不注入游戏进程
- 仅读取游戏客户端公开的 LCU API
- 截图仅在本地处理，不上传服务器

## 6. 技术架构

### 6.1 技术栈
- **框架**: Electron + Vue 3
- **UI库**: ShadcnVue + TailwindCSS
- **截图**: screenshot-desktop
- **OCR**: Tesseract.js（本地识别，无网络依赖）
- **图像处理**: sharp / jimp（图像预处理、裁剪）
- **特征检测**: OpenCV.js 或 自定义颜色/轮廓检测算法

## 7. 状态流转

### 7.1 应用状态
```
IDLE           → 空闲状态，等待配置
CONFIGURED     → 已配置，等待启动监控
MONITORING     → 监控中，轮询英雄选择
CHAMPION_FOUND → 检测到英雄，弹窗显示
GAME_ACTIVE    → 游戏进行中，截图服务启动
AUGMENT_SELECT → 海克斯选择界面，浮窗显示
```

### 7.2 状态转换
```
IDLE ──配置路径──▶ CONFIGURED ──启动监控──▶ MONITORING
                                              │
                               检测到英雄ID ◀──┘
                                              │
                                              ▼
                                        CHAMPION_FOUND
                                              │
                               检测到游戏窗口 ◀──┘
                                              │
                                              ▼
                                        GAME_ACTIVE
                                              │
                               10秒后开始截图 ◀──┘
                                              │
                               识别到海克斯界面
                                              │
                                              ▼
                                       AUGMENT_SELECT
                                              │
                               用户选择后 ────▶ GAME_ACTIVE
                               游戏结束 ─────▶ MONITORING
```

## 8. 边界情况处理

### 8.1 异常情况

| 场景 | 处理方式 |
|------|----------|
| LCU 未启动 | 显示提示，等待用户启动游戏客户端 |
| 游戏路径无效 | 提示用户重新配置 |
| OCR 识别失败 | 静默重试，不显示错误浮窗 |
| 网络问题 | 使用本地缓存数据 |
| 多开游戏 | 只监控第一个实例 |

### 8.2 游戏生命周期

| 阶段 | 系统行为 |
|------|----------|
| 大厅 | 空闲监控 |
| 选人阶段 | 检测英雄ID，显示弹窗 |
| 加载中 | 启动截图服务倒计时 |
| 游戏中 | 定时截图，识别海克斯 |
| 游戏结束 | 停止截图，回到监控状态 |
| 下一局 | 重复上述流程 |

## 10. 附录：图像特征检测算法详解

### 10.1 海克斯界面检测流程

```
┌─────────────────┐
│   原始截图      │
└────────┬────────┘
         │
         ▼
┌─────────────────┐
│  缩放到标准尺寸 │ ← 统一处理不同分辨率
└────────┬────────┘
         │
         ▼
┌─────────────────┐
│  提取中央区域   │ ← 海克斯卡片总在屏幕中央
└────────┬────────┘
         │
         ▼
┌─────────────────┐
│  颜色空间转换   │ ← RGB → HSV
└────────┬────────┘
         │
         ▼
┌─────────────────┐
│  边框颜色检测   │ ← 检测紫色/金色/棱彩边框
└────────┬────────┘
         │
         ▼
┌─────────────────┐
│  轮廓提取       │ ← 寻找矩形轮廓
└────────┬────────┘
         │
         ▼
┌─────────────────┐
│  卡片验证       │ ← 验证3张卡片的位置和尺寸
└────────┬────────┘
         │
    是否检测到?
    ┌────┴────┐
    │是       │否
    ▼         ▼
┌───────┐ ┌───────┐
│进行OCR│ │跳过   │
└───────┘ └───────┘
```

### 10.2 颜色阈值参考（HSV空间）

| 稀有度 | 边框颜色 | H范围 | S范围 | V范围 |
|--------|----------|-------|-------|-------|
| 银色 Silver | 灰白色 | 0-180 | 0-30 | 180-255 |
| 金色 Gold | 金黄色 | 20-40 | 150-255 | 180-255 |
| 棱彩 Prismatic | 紫色渐变 | 130-160 | 100-255 | 150-255 |

### 10.3 卡片位置估算（1920x1080基准）

```
屏幕宽度: 1920px
屏幕高度: 1080px

卡片区域:
- 左卡片: x=340, y=180, w=280, h=420
- 中卡片: x=820, y=180, w=280, h=420
- 右卡片: x=1300, y=180, w=280, h=420

名称区域（相对于卡片）:
```

### 10.5 名称模糊匹配

由于 OCR 可能存在识别误差，需要进行模糊匹配：

```javascript
// 使用 Levenshtein 距离进行匹配
function findBestMatch(ocrResult, augmentNames) {
  let bestMatch = null;
  let minDistance = Infinity;

  for (const name of augmentNames) {
    const distance = levenshtein(ocrResult, name);
    const similarity = 1 - distance / Math.max(ocrResult.length, name.length);

    if (similarity > 0.7 && distance < minDistance) {
      minDistance = distance;
      bestMatch = name;
    }
  }

  return bestMatch;
}
```
