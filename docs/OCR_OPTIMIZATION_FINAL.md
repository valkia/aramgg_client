## 🔧 关键技术

### 1. 精准裁剪策略

```javascript
// 旧方案：裁剪 30%-70% 区域（包含大量 IDE/终端干扰）
const cropX = Math.round(width * 0.3)
const cropY = Math.round(width * 0.3)
const cropWidth = Math.round(width * 0.4)
const cropHeight = Math.round(height * 0.4)

// 新方案：聚焦卡片名称区域
const cropX = Math.round(width * 0.25)      // 25% 开始
const cropY = Math.round(height * 0.4)      // 40% 开始（名称位置）
const cropWidth = Math.round(width * 0.5)   // 50% 宽度（3张卡片）
const cropHeight = Math.round(height * 0.15) // 15% 高度（只要名称）
```

### 2. 图像预处理

```javascript
const croppedBuffer = await sharp(imageBuffer)
    .extract({ left: cropX, top: cropY, width: cropWidth, height: cropHeight })
    .greyscale()              // 灰度化（提高文字识别）
    .normalize()              // 自动归一化对比度
    .linear(1.5, -(128 * 0.5)) // 增强对比度
    .toBuffer()
```

**效果对比**:
- 旧方案识别: `党 光 ) | 避` (乱码)
- 新方案识别: `台风 帽 上 加 帆 急救 用 具` (清晰)

### 3. 模糊匹配算法

使用编辑距离（Levenshtein Distance）允许少量 OCR 错字：

- **≤2 字海克斯**: 必须精确匹配（避免误匹配）
- **3 字海克斯**: 允许 1 个字符错误
- **≥4 字海克斯**: 允许 `floor(len/3)` 个错误

**示例**:
- OCR: `帽上加帆` → 匹配到 `帽上加帽` (1个错误)
- OCR: `冰霜幽灵` → 精确匹配 `冰霜 幽灵`

### 4. 黑名单机制

过滤常见描述性词汇：

```javascript
const blacklist = new Set([
    '伤害', '护甲', '攻击', '防御', '生命', '法术', '魔法',
    '技能', '冷却', '移速', '暴击', '吸血', '穿透',
    '功能', '能力', '效果', '被动', '主动', '额外',
    '大力', '溢流', '砍伤', // 常见误匹配的短名称海克斯
])
```

尽管最终达到 100% 识别率，但必须承认：

- **中文识别能力一般** - 需要大量图像预处理和后处理
