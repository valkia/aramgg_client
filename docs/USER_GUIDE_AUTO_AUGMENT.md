# 自动海克斯检测使用指南

## 功能概述

应用现在可以在游戏中**自动检测海克斯选择**，无需手动按 F1！

当你在游戏中看到海克斯选择界面时，应用会：
1. 🎯 自动识别 3 张海克斯卡片
2. 📊 显示每个海克斯的胜率和推荐度
3. ⚡ 实时更新，响应速度 < 0.5 秒

---

## 使用流程

### 1. 启动应用并配置游戏路径
```
启动应用 → 设置 → 选择游戏路径
或者让应用自动检测路径
```

### 2. 进入游戏
```
启动英雄联盟客户端
进入匹配/排位
```

### 3. 应用自动监控游戏阶段
```
✅ 检测到游戏开始（GameStart）
✅ 检测到游戏进行中（InProgress）
   → 自动启动截图服务（每 200ms 截图一次）
```

### 4. 海克斯选择时自动识别
```
游戏中出现海克斯选择界面
  ↓
应用自动识别卡片（约 200-500ms）
  ↓
弹出窗口显示推荐
```

---

## 两种使用方式

### 方式 1: 全自动（推荐）
- ✅ 无需手动操作
- ✅ 进入游戏后自动启动
- ✅ 海克斯出现时自动识别
- ✅ 实时显示推荐

**适用场景**: 日常游戏，快速决策

### 方式 2: 手动触发（保留）
- 按 **F1** 键手动截图
- 立即分析并显示结果
- 可在任何时候使用

**适用场景**: 需要重新查看，或自动识别失败时

---

## 识别条件

应用会在以下条件下触发通知：

| 条件 | 说明 |
|------|------|
| 检测到 3 张卡片 | 必须识别到完整的 3 张海克斯 |
| 置信度 > 70% | 确保识别准确性 |
| 卡片间距均匀 | 验证是海克斯选择界面 |
| 位置在屏幕中央 | 排除其他游戏元素 |

### 检测精度

## 常见问题

### Q1: 为什么没有自动识别？
**可能原因**:
1. 游戏阶段检测失败 → 检查日志是否有 `InProgress` 阶段
2. 卡片识别不准确 → 检查日志中的置信度
3. 截图服务未启动 → 检查是否有自动截图日志

**解决方法**:
- 手动按 F1 触发识别
- 重启应用
- 检查游戏路径配置

### Q2: 截图是全屏的？
**说明**: 这是正常的
- 应用截取整个屏幕以确保可靠性
- 分析算法会自动定位游戏区域
- 不影响识别准确性

### Q3: 识别速度慢？
**检查**:
- 查看日志中的 `Captured in XXms`
- 正常应该 < 150ms
- 如果 > 500ms，可能是系统性能问题

### Q4: 如何关闭自动识别？
**方法 1** (临时): 不进入游戏
**方法 2** (配置):
```javascript
// 在开发者工具中执行
window.ipcRenderer.invoke('auto-screenshot-set-config', {
    enableAnalysis: false
})
```
