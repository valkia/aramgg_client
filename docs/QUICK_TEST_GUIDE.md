# 快速测试指南

## 30 秒快速开始

### 1. 运行测试

```bash
npm run test:screenshots
```

### 2. 查看结果

程序会自动测试最近的 20 张截图，显示统计报告：

```
📊 测试统计报告
总测试数: 20
高置信度检测: 0 (0.0%)
部分检测: 0 (0.0%)
未检测: 20 (100.0%)
分析失败: 0 (0.0%)
平均耗时: 75ms
```

## 理解输出

### 四种检测结果

| 状态 | 含义 | 是否正确 |
|------|------|--------|
| ✅ **高置信度检测** | 3 张有效卡片，验证通过 | **是** |
| ⚠️ **部分检测** | 卡片不足或验证失败 | **有问题** |
| ❌ **未检测** | 没有找到有效卡片 | **是**（对于非海克斯界面） |
| ❌ **分析失败** | 程序错误 | **不对** |

## 测试场景

### 场景 A: 验证修复成功（当前情况）

**情形**: 已有的 104 张截图都是菜单或空屏幕

**预期结果**:
```
高置信度检测: 0 (0.0%)
未检测: 20 (100.0%)  ✅ 说明没有误检
```

**结论**: 修复成功！不会再有虚假的海克斯检测

### 场景 B: 测试真实海克斯截图

**目标**: 验证在真实游戏中是否能正确检测

**步骤**:

1. **启动游戏**
   ```
   启动英雄联盟客户端
   ```

2. **进入海克斯阶段**
   ```

   ```
   按 F1 或自动截图会保存到:
   C:\Users\du\.lol-tips-client\screenshots\

   ```bash
   npm run test:screenshots
   ```

5. **查看结果**
   ```
   应该在报告中看到:
   ✅ 高置信度检测的截图:
     1. screenshot-XXXX.png - [金色, 紫色, 银色] - 95.0%
   ```

**成功指标**:
- 高置信度检测比例 > 80%
- 所有海克斯截图都被正确识别
- 没有虚假正例（部分检测为 0）

## 常见情况

### 情况 1: 所有截图都是"未检测"

**原因**: 截图中没有海克斯选择界面

**说明**:
- ✅ 正常现象（说明误检已修复）
- 需要进行真实游戏截图来测试

### 情况 2: 出现"部分检测"

**可能原因**:
1. 海克斯卡片太小或太大
2. 卡片颜色不清晰（光线问题）
3. 卡片不在屏幕中央
4. 卡片间距不均匀

**对策**:
- 重新进行游戏，确保光线充足
- 确保卡片处于标准位置
- 再次运行测试

### 情况 3: 出现"分析失败"

**原因**: 文件损坏或程序错误

**对策**:
- 检查截图文件是否有效
- 运行 `npm run build` 重新构建
- 重新运行测试
